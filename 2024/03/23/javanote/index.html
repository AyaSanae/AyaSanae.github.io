Collection

  graph TD;
      Collection-->List;
      Collection-->Set;
      List-->ArrayList;
      List-->LinkList;
      Set-->HashSet;
      Set-->TreeSet;
      HashSet-->LinkHashSet;

List系列集合:添加的元素是有序,可重复,有索引的.
Set系列集合:添加的元素是无序,不可重复,没索引的.

Collection是单列集合的祖宗接口,它的功能在所有单列集合都可以继承使用.

Collection常用方法:

add();方法对于List集合来说会永远返回true,而对于Set系列集合来说会检查是否有重复有的话则返回false.
Collection里的remove();方法只能通过对象的形式删除而不能通过下标,因为是共性的.要是删除的对象不存在则会返回false.

contains()

该方法的在底层依赖equals();方法--依赖地址值判断,所以如果集合里面装的如果是自定义类型,
那就要在对应的javabean中重写equals();
如果没有在对应的javabean里重写equals();则会默认调用Object里的equals();

迭代器在Java类中是Iterator,迭代器是集合的专用遍历方式.

  graph TD;

Iterator注意事项

如果当前位置没有元素还要强行获取就会报NoSuchElementExpection
迭代器遍历完毕指针不会复位
循环中next()方法只能调用一次,多次调用会导致指针位置多次被移动,所以应该一个hasNext和一个next方法配套
迭代器遍历时,不能用集合的进行元素的增加或删除,只能使用迭代器的方法来进行,不然会报并发修改异常

List系列

Collection的方法List集合都继承了
List集合因为有索引所以多了很多索引操作方法.

List系列集合独有方法

remove方法

对于remove(1);这样的用法,会移除下标为1的元素,而不会移除1这个对象.
因为调用方法的时候,如果出现了重载现象,优先调用实参和形参类型一致的那个方法.
而remove(1)的用法,不会使得1自动装箱.所以如果要移除1,则要手动装箱.

列表迭代器

除了Collection有的迭代器,增强for,lambda表达式,这三种以外,
还可以用普通for循环--因为有下标.
还有一种List特有的--列表迭代器.
用法和迭代器一样,但是多了add()方法,而迭代器只有remove().
在遍历时要删除元素可以用迭代器,要添加元素可以用列表迭代器.

ArrayList底层原理

1.利用空参创建的集合,在底层创建一个默认长度为0的数组.

2.添加第一个元素时,底层会创建一个新的长度为10的数组.

3.存满时会扩容1.5倍.

4.如果一次添加多个元素,1.5倍放不下,则新创建数组的长度以实际为准.

泛型

泛型是JDK5中引入的特性,可以在编译阶段约束操作的数据类型,并进行检查
泛型的格式:<数据类型>

为什么要有泛型?

那就要先提到没有泛型的时候,集合如何存储数据
如果我们没有给集合指定类型,默认认为所有数据类型都是Object类型,此时可以往集合里
添加任意的数据类型.
这样就回带来一个坏处:我们获取数据的时候无法使用他的特有行为--多态弊端.

这时候如果有泛型的话就可以把类型进行统一.而且我们获取数据的时候也可以不强转,非常
方便.

所有泛型有统一数据类型,避免强制类型转换可能会出现的异常的好处.

Java中的泛型是伪泛型.

Java源文件:
ArrayListlist = new ArrayList<>();
Class文件:
ArrayList list = new ArrayList();

以上过程叫做泛型的擦除.

泛型的细节

泛型只支持引用数据类型

指定泛型的具体类型后,传递数据时可以传入该类型或其子类类型

如果不写泛型默认类型是Object

泛型的通配符

先来看一段需求:定义一个方法,方法形参是集合,但集合中的数据类型不确定.


但是此时有点小问题,他可以是任意的,E可以是任意数据类型(除了基本数据类型),
如果我们想把E限定在一定的范围内要怎么做?

此时可以使用泛型通配符:
?表示不确定的类型
他可以进行类型限定
? extends E:表示可以传递E或E的所有子类类型
? super E:表示可以传递E或E的所有父类类型

泛型通配符应用场景

关键点:可以限定泛型范围

1.如果我们在定义类,方法,接口的时候,如果类型不确定,就可以定义泛型类,泛型方法,泛型接口.
2.如果类型不确定,但是能知道以后只能传递某个继承体系中的,就可以使用泛型通配符.
