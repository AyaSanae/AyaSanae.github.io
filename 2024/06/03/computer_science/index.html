存取方式

随机存取存储器(Random Access Memory)RAM,读写任何一个存储单元所需要的时间都相同,与存储单元所在的物理位置无关.

串行的
顺序存取存储器(Sequential Access Memory)SAM,读写一个存储单元所需的时间取决于存储单元所在的物理位置.如:磁带

串行的
直接存取存储器(Direct Access Memory)DRM:即有随机存取的特性,也有顺序存取特性,先直接选取信息所在区域然后按顺序方式读取.如:HDD

相联存储器(Associative Memory)即可以按内容访问的存储器,(Content Addressed Memory,CAM),可以按照内容检索到存储位置进行读写,如:快表

速度上:
AM > RAM > SAM > DAM

信息的可更改性

读写存储器(Read/Wire Memory)即可读也可写的存储器,如:磁盘,内存,Cache
只读存储器(Read Only Memory)只能读,不能写(如:CD,Blue-Ray等)
事实上,现在的很多的ROM都可写.

存储器性能指标

存储容量: 存储字数 * 字长(如:1M * 8位)
单位成本: 每位价格 = 总成本/总容量
存储速度: 数据传输率 = 数据的宽度/存储周期

1.存取时间(Ta):存取时间是指从启动一次存储器操作到完成该操作所经历的时间,分为读出时间和写入时间,
2.存取周期(Tm):存取周期又称为读写周期或访问周期,它是指存储器进行一次完整读写所需要的全部时间,即连续两次独立地访问存储器操作,(读或写操作)之间所需的最小时间间隔

主存带宽(Bm):主存带宽又称为数据传输率,表示每秒从主存进出信息的最大数量,单位为字/秒,字节/秒,位/秒

一个字等于多少字节?

一个字等于多少个字节，与系统硬件（总线、cpu命令字位数等）有关。总线一般被设计来传输固定大小的一块数据，这块数据被称为字-word
1字节（byte） = 8位（bit）
在16位的系统中（比如8086微机） 1字 （word）= 2字节（byte）= 16（bit）
在32位的系统中（比如win32） 1字（word）= 4字节（byte）=32（bit）
在64位的系统中（比如win64）1字（word）= 8字节（byte）=64（bit）

DRAM的刷新

为了解决DRAM栅极电容带来的缺点,我们给DRAM的电容进行刷新操作.
那么:
刷新周期: 2ms
每次刷新数量: 以行为单位,每次刷新一行存储单元

之所以以行为单位来刷新,是因为如果存储单元数量巨大,比如1M,那就有1M根选通线从译码器连出,工程量巨大,效率低下.
如果我们更改存储单元排列结构,将一维排布的存储单元转化为二位排列

使用两个译码器,一个行地址译码器,一个列地址译码器,这样可以优化存储单元排布。
一维的方案,译码器要处理 2^20 (20根地址线)的信息,转到二维后每个译码器只需要处理2^10的信息,即1k根选通线

一维转成二维后,对地址的解释也会有所变化,比如对于一维来说,0000 0000,表示的是第0个存储单元,而对于二维来说,前四个0表示行后四个表示列,只有被行,列地址同时选中的存储单元才能被读写.

刷新方法:这里且不讨论硬件实现,读出一行的信息后重新写入,占用1个读/写周期
在什么时刻刷新?
假设DRAM的内部结构排列成128 * 128,存取周期0.5us,那么2ms的极限时间就有2ms/0.5us = 4000个周期
也就是说2ms可以执行4000个周期的读写操作

思路一:每次读写完都刷新一行,也就是说,每次读写,用0.5us读写,0.5us刷新,系统的存取周期变1us--分散刷新


那么在2ms内我们可以刷新2ms / 1us = 2000次,而我们的DRAM阵列假设只有128行所以绰绰有余.
看起来很低效

思路二:2ms内集中安排时间全部刷新,那么系统的存取周期还是0.5us,2ms内划出一段时间来刷新,这个段时间无法访问存储器,这段时间被称为访存死区 -- 集中刷新


因为只有128行,所以我们只需要留下128个读写周期
看起来也好不到哪去

思路三:2ms内每行刷新一次即可,2ms内需要产生128次刷新请求,我们可以每隔2ms/128 = 15.6us一次,也就是说,每15.6us会有0.5us的死区 -- 异步刷新


而且我们可以选择在译码阶段我们进行刷新
看起来高效了很多

DRAM的地址线复用技术

前面提到,我们把一维的存储单元排布转换成了二维.

寻址时,如果我们同时把行地址列地址同时送到行地址译码器和列地址译码器,那么我们便需要相对应的地址线.比如0000 0000这一个地址会同时送到行地址译码器,列地址译码器,也就是说地址线要8根.

也就是说,我们的地址有多少位,我们就需要多少根地址线,这看起来理所当然,但我们有更好的方案,如果地址线一多,这种一次送到的方法就会显得鸡肋了.

我们可以先传行地址再传列地址或者反过来,这样如果是8位地址,我们每次只需要传4位,也就是地址线数量减半,芯片引脚地址线减半

SRAM的容量较小,所以送地址的时候都是同时送,也就是不分批.

ROM

Read Only Memory--只读存储器,断电后能持续保存数据

MROM(Mask Read-Only Memory) --掩模式只读存储器
厂家按照客户需求,在生产过程中直接写入信息,之后任何人不能重写,可靠性高,灵活性极低,生产周期长


PROM(Programmable Read-Only Memory) --可编程只读存储器
用户可用专门的PROM写入器来写入信息,一旦写入不可更改


EPROM(Erasable Programmable Read-Only Memory) --可擦可编程只读存储器
允许用户写入信息,并用特定方式擦除数据,可以多次重写


UVEROM(Ultraviolet rays Read-Only Memory) --可擦可编程只读存储器
允许用户写入信息,用紫外线照射一定时间来擦除数据.


EEPROM(Electrically Erase Programmable Read-Only Memory) --电可擦除可编程只读存储器
可用电擦除,擦除特定字


Flash Memory -- 闪速存储器
由EEPROM发展而来,可进行多次快速擦除重写,像是u盘,SD卡
由于闪存要先擦再写,所以闪存的读会比写快得多.


SSD(Solid State Drives) --固态硬盘
由存储单元(Flash)+控制单元构成,可进行多次快速擦写.

PC主板上的BIOS程序写在ROM里,逻辑上主存由RAM + ROM组成,而且常常统一编址,开机后先读取BISO程序.

双端口RAM

先看回存储周期:

DRAM的恢复时间比较长,有可能是存取的几倍.
虽然存取的时间很快,但是存取完要相当长的恢复时间,对于多核CPU来说对性能削弱是毁灭性的.

为了实现双端口RAM,我们要有两组完全独立的数据线,地址线,控制线.CPU,RAM里也要有更复杂的控制电路

CPU1,CPU2对双端口RAM访问的可能情况:
1.两个端口同时对不同的地址存取数据 √
2.两个端口同时对同一地址读出数据 √
3.两个端口同时对同一地址写入数据 ×
4.两个端口对同一地址单元,一个写入,一个读出 ×

解决方法: 使用"忙"信号,由判断逻辑决定暂时关闭一个端口 -- 即被延时访问

感觉很低效.

多体并行存储器

每一列可以看成一根内存

每个模块都有相同的容量和存取速度,各模块都有独立的读写控制电路,地址寄存器和数据寄存器,他们既能并行工作又能交叉工作

高位交叉编址


CPU是通过地址来向访存的,高位交叉编址是指,在地址最高位留出一定的位置来区分访问哪一根内存,
图中有四个内存,假如8位地址,那么可以留出最前面的两位来确定访问哪一个根内存.

假设每条内存有8个存储单元,那么有4 * 8 = 32个存储单元,也就是2 ^ 5个,也就是可以用5bit来作为地址,
那么对于第一根内存条有00 000,后面三个000表示体内地址

可以看出,地址编排方式是竖着编的

假如说,每个存储体存储周期为T,存取时间为r假设T = 4r,那么恢复时间为3r.
对于高位交叉编址,连续访问:00000,00001,00010,00011,00100

可以看到压力全给到了M0内存
可以算出连续存取n个字,耗时nT,这里是5T.

低位交叉编址

同高位交叉编址,不过是用最低位来表示第几根内存

可以看出,地址编排方式是横着编的

假如说,每个存储体存储周期为T,存取时间为r假设T = 4r,那么恢复时间为3r.
对于低位交叉编址,连续访问:00000,00001,00010,00011,00100

可以看到负载均匀,在存取00000后,直接去存取00001不用等待M0的恢复,只花了r时间,如此往下,
M3开始,M0刚好恢复,所以可以直接存取00100.有多线程的味道了.

图里划虚线的地方刚好是T,如果考虑上最后的恢复时间,那么总耗时是T + 4r
如果只考虑存取的总用时,那么只有5r

那么可以得出通用计算公式: 连续取n个存储字 耗时 = nr + 3r = T + (n -1)r (包含了最后恢复时间)
根据公式来看,当连续读写的字越多,时间越接近r

可以看出低位交叉编址比高位交叉编址高效的多的多.

为什么是"连续访问"
不碰到跳转语句的话,程序是一句一句顺序的执行的,亦或者像是数组这种结构数据紧密相连.

采用"流水线“(看看上面存储体时间图),看起来宏观上是并行的,微观上是串行的.宏观上,一个存储周期内,m体交叉存储器(m体指的是m个存储体,比如m条内存条),可以朝外提供的数据量为单个模块的m倍

存储周期为T,存取时间为r(或总线传输周期为r -- 指cpu通过数据总线传输数据至少要r的时间)
,为了使得流水线不间断,我们应该保证模块数m(内存条数) > T / r
原因如下:

如果给出一个地址x,我们可以通过体号(低位的那几个)来判断在那个存储体,也可以直接用x % m,m指的是存储体总量.

双通道内存使用的就是低位交叉的多体存储器

单体多字存储器

多体并行存储器的合并版.

整个存储器只有一套读写电路
本来每次只能读写一个字,但是到了这里的单体多字存储器一次能读4个字也就是一行.

显然,一次只能读4个字就显示出了他的鸡肋 -- 不能单独取其中的某个字,假如数据存在第一行后面两个字和第二行前面两个字,那么要读两次,而且会出来4个字的无效数据,而且指令和数据在主存内都是连续存放的.

但是吧,单体多字存储器确实提升了速度,每一个存取周期能读4个字

磨损均匀技术

得益于闪存翻译层的地址映射,如果一个block被擦除的次数过多,我们可以把这个块的数据迁移到新的block,而旧的block用来存一些基本不擦除的数据,比如视频等

动态磨损均匀: 写入数据时,优先选择擦除次数少的新闪存block

静态磨损均匀: SSD检测并自动进行数据分配,迁移,让老旧的闪存block承担读为主的存储任务,新闪存block承担更多的写任务.

来看个有意思的题目:
某固态硬盘采用磨损均衡技术，大小为2^40 B=1TB，闪存块的擦写寿命只有210=1K次。某男子平均每天会对该固态硬盘写2^37 B=128GB数据。在最理想的情况下，这个固态硬盘可以用多久?

得益于磨损均匀技术，每个block擦除次数被均衡
1TB / 128G = 8
也就是说平均8天每个块都被擦除一次
而每个闪存能擦1k,所以要8k天,即23年.

Cache的工作原理

我们知道程序指令是一行一行执行的,也是一行一行存储的.
而我们运行程序的时候,也并不是要把程序的全部指令加载进内存.
比如微信,往往我们使用的时候,都只用到了其中的几个功能如:聊天,朋友圈.
当相关的指令被加载进内存,CPU便可从内存中读指令运行.

前面也说了,往往我们不会去使用多个功能板块;而且程序有局部性原理.
所以我们可以将特定的板块复制到高速的Cache,
让CPU和Cache直接交互,速度瓶颈就不明显.

这样Cache在CPU与内存之间就相当于充当缓冲区了,这也是为什么Cache被称为高速缓存


Cache被集成在CPU中,并使用SRAM实现,往往有多级,比如L1,L2,L3缓存,不同级速度,大小也不一样

性能分析


设tc为访问一次Cache耗时,tm为访问主存耗时
我们把CPU想要访问的信息已经在Cache中的比率叫命中率H(Hit),就是Cache里有多大可能有CPU要用的数据.
那么未命中的概率也就是缺失率: M(Miss) = 1 - H

那Cache-主存的平均访问时间t = H * tc + (1 - H) * (tc + tm)
好好想想,这不难理解.加号前半段是在Cache里找到了,后面是在Cache里没找到跑主存去找了

当然我们的CPU可以同时在Cache和内存里找,Cache命中后立即中止在内存中的查找,则
平均访问时间t = H * tc + (1-H) * tm

来看这个:假如Cache速度是主存5倍,命中率为95%,则用Cache后,存储器性能提升多少(设Cache和主存同时访问,若Cache命中则中断访问主存),如果先访问Cache再访问主存呢?

那我们可以

设Cache存取周期t,则主存存取周期5t.
也就是说,Cache如果命中访问时间为t,未命中为5t.
则平均访问时间: 0.95 * t + 0.05 * 5t = 1.2t
而原本主存存储周期为5t,所以快了5t / 1.2t ≈ 4.17倍

如果先访问Cache再访问主存:
平均访问时间: 0.95 * t + 0.05 * (t + 5t) = 1.25t
也就是5t / 1.25t = 4倍


可以看出无论是哪种访问方式,Cache的加入都可以获得十分可观的收益.

Cache和主存的映射方式 -- Cache与主存的数据对应关系

给每个Cache块增加一个标记,记录对应的主存块号.一开始直接全部初始化为全0.
但是这样是不够的,因为这相当于所有Cache块都存的0号内存块的信息.
所以我们需要个开关 -- 有效位

全相联映射

主存块可以放在Cache的任意位置

假设某计算机的主存地址空间大小位256MB,按字节编址,其数据Cache有 8 Cache行(即Cache块),行长64B

8Cache行,也就是 8 * 64B = 512B 即Cache总大小
256M即2^28,也就是说主存地址有28位
64B的Cache行长的话就是2^6,那我们就可以划 2^28 / 2^6 = 2^22


OK,那我们的CPU要怎么访存?这个比较简单:

对于要访存的地址,我们检查主存块号,这里是前22位,看看Cache里有没有,有的话看有效位是否为有效,有效则命中.然后根据块内地址,这里是6位,找到相对应的字节就行.


如果没命中或者有效位为0,则正常访问主存

直接映射

每个主存块只能放到一个特定的位置
Cache块号 = 主存块号 % Cache总块数

如果取余后要放的位置已经有数据了,则直接覆盖.并且修改标记为(主存块号).
这就会导致0...0000 会和 1...000放到同一块Cache块,而不是放到空闲的位置,造成性能浪费

虽然不能解决空闲位置浪费问题,但我们可以优化这个方案:
对于Cache块号 = 主存块号 % Cache总块数,我们上面的例子Cache总块数是8,也就是2^3 也就是:
Cache块号 = 主存块号 % 2 ^ 3 ,也就是相当于留下主存块号的最后三位二进制数.
即,对于Cache总块数为2 ^ n 的时候,主存块号的末尾n位直接反映它在Cache中的位置

也就是说,对于我们的例子,我们的22bit主存块号,可以直接拿出3bit来表示Cache行号
通过检查主存块号的最后三位就可以确定要放到Cache的哪一行.
剩下的前面的19bit可以用来当标记.

那么我们的CPU要怎么访存?
比如访问: 0...01000 001110
1.根据主存块号后3bit确定要放在哪一Cache行.这里是000即0行
2.若主存块号的前19bit和Cache的标记匹配且有效位为1,则Cache hit
3.根据后地址6bit来确定访问哪一个存储单元,这里是001110
4.如果Cache miss,则正常访问主存.

这样优化的好处是不用计算取余,电路设计也很简单.

组相联映射

Cache块分为若干组,每个主存块可以放到特定分组中的任一位置
组号 = 主存块号 % 分组数
算出组号后哪里组内哪里有空位放到哪

这我们的例子中,我们的Cache可以分为4组,2块一组,这种分法叫2路组相连映射.

如图,绿色和橙色块都被放到第一Cache组,哪有空位就放哪.

同直接映射的优化,这里涉及了取余,故:
Cache块有2 ^ n 的情况下,我们的主存块号最后n位可以表示要放到哪个Cache组
这里是主存块号最后2bit
然后把22bit主存块号拆开,剩下20bit当标记位

CPU如何访存?
如: 1...1101 001110
1.根据主存块号最后2bit确定在哪一Cache块组

近期最少使用算法 -- LRU

Least Recently Used - LRU
为每一个Cache块设置一个计数器,记录每个Cache块已经多久没被访问,当Cache满了后,替换计数器中最大的.

对于计数器的规则:
1.命中时,所命中的Cache行计数器清零,比其低的计数器加1,其余计数器不变.
2.未命中而且有空闲行的时候,新装入的Cache行计数器置0,其余非空闲行全加1.
3.未命中且无空闲行,计数器最大的Cache行被淘汰,新装入的Cache行计数器置0,其余全加1.

然后替换,清0对应的计数器,其他全加1.

第八次访存,1号主存块Cache hit,我们清空0号Cache行的计数器,然后让比0号Cache行计数器小的加一,这里是1,2号Cache行的计数器.

而且LRU算法实际运行效果优秀,Cache命中率高.足够优雅.

但是吧,如果被频繁访问的主存块数量 > Cache行数量,则可能发生抖动

最不频繁使用算法 -- LFU

Least Frequently Used -- LFU
为每个Cache块设置一个计数器,用于记录每个Cache块被访问几次,当Cache满了后替换计数器最小的

设计理念和LRU相似,注意不要混淆.
LFU是从访问频率入手,LRU是从闲置时间入手.

对于新调入的块,计数器置0,之后每被访问+1,所有Cache行满了后,替换计数器值最小的一Cache行.
如果有多个计数器最小行,则可以按照FIFO或者行号递增来进行淘汰.


还是比较简单的.

看似优雅实则鸡肋.

写回法 -- write-back

当CPU对Cache写命中时,只修改Cache内容,不同步更新主存,只有当该块被换出才写回内存

方法也很简单,我们只需要在Cache中新加个脏位来表示这一Cache行有没有被写过.
通过标记位来确定回写到哪.

全写法 -- write-through

写直通法,当CPU对Cache写命中,必须同时写入Cache和主存,一般使用写缓冲(write buffer)


写缓冲用SRAM实现,采用FIFO队列.写缓冲的引入缓和了主存的速度不住,使得这种方法变得更为优雅
写缓冲有独立的电路使得缓冲里的数据写回主存.

所以我们不需要脏位,淘汰时也不需要回写
如果读写操作不频繁的话效果很好,但是如果很频繁的话,可能导致写缓冲饱和而发生阻塞.

写不命中
写分配法 -- write-allocate

当CPU对Cache写不命中时,把主存的块调入Cache,在Cache中修改,通常搭配写回法.

非写分配法

当CPU对Cache写不命中时只写入主存,不调入Cache,通常搭配全写法

指令格式分类

地址码数目

零地址指令

1.不需要操作数,如空操作,停机,关中断等指令.
2.堆栈计算机,两个操作数隐含存在栈顶和次栈顶,计算结果压回栈顶.

一地址指令

格式: OP + A1 (操作数+一个操作数,操作数指出指令执行的操作所需要数据的来源)
1.只需要单操作数,如加1,减1,取反,求补等
含义: OP(A1) -> A1 (取出A1地址指向的数,根据操作数执行计算操作,结果放回A1,(A1)表示A1所指向的内容),三次访存: 取地址 -> 读A1 -> 写A1

2.需要两个操作数,但其中一个操作数隐含在某个寄存器(如ACC)
格式:（ACC)OP(A1)->ACC (取出ACC,A1,根据操作数执行计算操作,结果放回ACC)
两次访存: 取址 -> 读A1

二地址指令 && 三地址指令

格式: OP+ A1 + A2
A1为目的操作数,A2为源操作数
指令含义: (A1)OP(A2) -> A1

常用于需要两个操作数的算术运算,四次访存: 取指 -> 读A1 -> 读A2 -> 写A1

三地址:
格式: OP + A1 + A2 + A3(结果)
指令含义: (A1)OP(A2) -> A3
四次访存: 取指 -> 读A1 -> 读A2 -> 写A3

四地址指令

格式: OP + A1 + A2 + A3(结果) + A4(下一地址)
指令含义: (A1)OP(A2) -> A3, A4 = 下一条将要执行指令的地址
四次访存: 取指 -> 读A1 -> 读A2 -> 写A3
执行指令后,将PC值修改为A4指向的地址.

指令长度

指令字长: 一条指令的总长度.(可能会变)
机器字长: CPU进行一次整数运算所能处理的二进制位数.(通常和ALU直接相关)
存储字长：一个存储单元中的二进制代码位数.(通常等于MDR)

我们可以用半字长,单字长,双字长指令来描述一条指令长度是机器字长的多少倍.
显然指令支撑会影响取指令所需要的时间,如:机器字长 = 存储字长 = 16bit,则一条双字长指令要两次访存.

定长指令字结构: 指令系统中所有指令的长度都相等.
变长指令字结构: 指令系统重各种指令长度不等.

操作码长度

定长操作码: 指令系统中所有指令的操作码长度都相等.
定长操作码的数量是有限的,是2^n条,这样的话控制器设计就会简单,但是明显,灵活性差.

可变长操作码:指令系统重的各指令操作码长度可变.

程序控制类

短码:更短的操作码
长码:更长的操作码
像是上图的二地址指令是相较于三地址指令是长码.

在设计扩展操作码指令格式时,必须注意:
1.不允许短码是长码的前缀,即短操作码不能与长操作码前面部分的代码相同.
很好理解,要是三地址的前4bit是1111,就会被认为是二地址.
2.各指令操作码不能重复

通常,对使用频率高的指令,分配较短的操作码,对使用频率较低的指令,分配较长的操作码,从而尽可能减少指令译码和分析时间.

1.15条三地址
2.12条二地址
3.62条一地址
4.32条零地址

我们只需要0000 - 1011,也就是12条,1011往后也就是1100,也就是二地址指令留出的.

3.对于一地址,开头一定是 1111 11,这样就不会和二地址指令重合,同时我们只需要留出最后4bit来表地址,62条的话:1111 11 000000 - 1111 11 11111110,也就是62条,留出1111 11 111111给零地址

顺序寻址

但要是我们的主存是按字节编址而不是按字编址呢? 那我们的PC就应该+2
如果用的变长指令字结构又要怎么办呢? PC加多少就取决于变长指令字的长度了.

所以对于指令寻址,并不是简单地让PC+1,但是可以确定的是PC始终存放了下一条指令的地址

直接寻址

指令字中的形式地址A就是操作数的真实地址EA,也就是EA = A

对于图中的访存:
取指令 -> 执行指令 2次

优点: 简单,指令执行阶段只需要访问一次主存,不需要专门计算操作数地址.
缺点: A的位数决定了该指令的操作数的寻址范围,操作数的地址不易修改.
假如我们的操作数只有16bit,那寻址能力只有0 -- 2^16 -1

间接寻址

指令的地址字段给出的形式地址不是真正的地址,而是操作数的有效地址所在的存储单元的地址,也就是操作数地址的地址,即EA = (A)

对于图中的访存:
取指令 -> 执行指令找真地址 -> 执行指令 3次

emmm,我认为你下面这张两次间接寻址图也能看懂,A1旁边的1代表不是真实地址,就不多说了.

优点:
可扩大寻址范围(有效地址EA的位数大于形式地址A的位数)
对于16bit的形式地址,因为他前面还跟着操作码,所以可以表示的只有0- 2 ^ 16 -1 个地址,
然而我们可以在A地址的主存位置上存满真实地址,这一定会比形式地址大,因为整个主存块都可以用于保存真实地址,而不用考虑操作码.

多次间接寻址有利于写程序,用间接寻址可以方便的完成子程序返回(想想函数间多次间多次调用然后按顺序出栈)

缺点:
指令在执行阶段要多次访存.

寄存器寻址

在指令字中直接给出操作数所在的寄存器编号,即EA = Ri,其操作数在由Ri所指的寄存器中

访存:
取指令 -> 执行指令 1次

优点:
快! 指令在执行阶段不访问主存,只访问寄存器,指令字短(因为寄存器不多)而且执行速度快,支持向量/矩阵运算.

缺点:
贵,寄存器贵,寄存器有限,所以寻址能力有限.

寄存器间接寻址

寄存器Ri中给出的不是一个操作数,而是操作数所在主存单元的地址,即EA = (Ri)

访存:
取指令 -> 执行指令(访问寄存器记录的主存地址) 2次

特点:
比一般间接寻址速度快,但最后还是要访存

隐含寻址

不是明显给出操作数地址,而是指令中隐含操作水地址

像图中,ACC里面值的地址并没有在地址码A中显式指出,而是默认已经在ACC里

优点:有利于缩短指令字长.
缺点:要增加存储操作数或隐含地址的硬件.

立即寻址

形式地址A就是操作数本身,又称为立即数,一般采用补码形式表示,#表示立即寻址特征.
格式: 操作码 + # + 立即数

请不要和直接寻址搞混,立即寻址的地址码的地方就是操作数,直接寻址还要根据地址去找操作数.

访存:
取指令 -> 执行指令(不需要再找了) 1次

优点:指令执行阶段不访问主存,指令执行时间最短
缺点:A的位数限制了立即数的范围,
如A的位数为n,且立即数采用补码时,可表示的范围为: -2^n-1 -- 2^n-1 -1
