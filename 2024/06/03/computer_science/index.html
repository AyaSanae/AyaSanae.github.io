现代计算机结构

0x00 存储设备

存储器的层次结构

按访问关系来看:
    flowchart LR
    CPU --高速--> Cache
    CPU --低速--> 主存
    Cache --高速--> CPU
    Cache --低速--> 主存
    主存 --低速--> Cache
    主存 --低速--> CPU
    主存 --更低速--> 辅存
    辅存 --更低速--> 主存

Cache一般由SRAM组成,成本更高,所以数量会更少,容量更低。

主存一般是DRAM,相对来说便宜不少,所以能数量更多容量更大,但是因为他的刷新等特性,
导致速度相较于SRAM来说要慢得多.

辅存诸如SSD,U盘,HDD等,速度与主存更是差更多,但得益于造价,可以有更大的容量,而且一般来说,都是能够持久性保存信息,不至于断电后丢丢失.

存取方式

随机存取存储器(Random Access Memory)RAM,读写任何一个存储单元所需要的时间都相同,与存储单元所在的物理位置无关.

串行的
顺序存取存储器(Sequential Access Memory)SAM,读写一个存储单元所需的时间取决于存储单元所在的物理位置.如:磁带

串行的
直接存取存储器(Direct Access Memory)DRM:即有随机存取的特性,也有顺序存取特性,先直接选取信息所在区域然后按顺序方式读取.如:HDD

相联存储器(Associative Memory)即可以按内容访问的存储器,(Content Addressed Memory,CAM),可以按照内容检索到存储位置进行读写,如:快表

速度上:
RAM > SAM > DAM

信息的可更改性

读写存储器(Read/Wire Memory)即可读也可写的存储器,如:磁盘,内存,Cache
只读存储器(Read Only Memory)只能读,不能写(如:CD,Blue-Ray等)
事实上,现在的很多的ROM都可写.

存储器性能指标

存储容量: 存储字数 * 字长(如:1M * 8位)
单位成本: 每位价格 = 总成本/总容量
存储速度: 数据传输率 = 数据的宽度/存储周期

1.存取时间(Ta):存取时间是指从启动一次存储器操作到完成该操作所经历的时间,分为读出时间和写入时间,
2.存取周期(Tm):存取周期又称为读写周期或访问周期,它是指存储器进行一次完整读写所需要的全部时间,即连续两次独立地访问存储器操作,(读或写操作)之间所需的最小时间间隔

主存带宽(Bm):主存带宽又称为数据传输率,表示每秒从主存进出信息的最大数量,单位为字/秒,字节/秒,位/秒

存储芯片的基本原理

为了时序协调工作与高效率控制,我们引入译码器来控制并行读写哪一个存储单元

我们给每一个存储单元编址,配合上译码器的话,译码器的n位地址可以表示2^n个存储单元.
那么总容量 = 存储单元个数 * 存储字长 这里是: 2^3 * 8bit = 8Byte
上图的译码器可以连接8个存储单元,输入接自MAR,MAR与地址总线连接

数据传输上,将通过MDR完成,MDR接入上图绿色线,并接入数据总线.
地址总线打开相对应的存储单元后,数据将由MDR流出到数据总线.
使用控制电路来使得MAR与MDR协调工作:

这样一来我们便可以往上抽象:

译码驱动是指,译码器的输出电信号不一定稳定,译码驱动则是稳定信号用的
读写电路便是前面图中绿色和红色的线.

结合以上的知识,如果我们有n条地址线接到MAR,那么说明就有2^n个存储单元
由此可以计算出总容量 = 存储单元个数 * 存储字长
有一种常见的描述存储芯片的说法:
8 * 8位的存储芯片
第一个8表示该芯片有8个存储单元
第二个8表示该芯片每个存储单元能存8位
8 * 8还可以写成 2^3 * 8bit也就是说,有三根地址线
#### 如何实现不同的寻址方式
先来看下面这张图

每个格子表示一个字节
那么每行就表示一个存储字,字长为4Byte
假设该存储矩阵总容量为1KB,那么就有256个字,1K个单元.

通常,现代计算机按字节编址,上图的存储矩阵我们假设了有1K个单元,那么便会有1K个地址,也就是说要十根地址线,通过10-1024的译码器.
虽然如此,现代计算机还是支持按字寻址,按半字,双字寻址的.

如果在按字节编址的前提下,我们想按字寻址,那么只需要将按字节寻址的地址左移两位.对于按字寻址来说,每个单元将是4个字节,矩阵中有256个单元.

一个字等于多少字节?

一个字等于多少个字节，与系统硬件（总线、cpu命令字位数等）有关。总线一般被设计来传输固定大小的一块数据，这块数据被称为字-word
1字节（byte） = 8位（bit）
在16位的系统中（比如8086微机） 1字 （word）= 2字节（byte）= 16（bit）
在32位的系统中（比如win32） 1字（word）= 4字节（byte）=32（bit）
在64位的系统中（比如win64）1字（word）= 8字节（byte）=64（bit）

SRAM && DRAM

Dynamic Random Access Memory -- DRAM 动态RAM
Static Random Access Memory -- SRAM 静态RAM

DRAM用于主存,SRAM用于Cache

存储元件不同导致的特性差异

SRAM通过双稳态触发器实现(想想D触发器),下面给出简略图:

这里不作详细探讨,有兴趣可以研究研究数电

与DRAM最大的不同的就是,电容比较鸡肋.
DRAM读出1时,电容会放电,电流流入数据线,导致电容里由1变0信息丢失,这种读出叫破坏性读出.而强大的双稳态触发器并不存在这种问题.
再生--这种解决方法,解决了DRAM读出造成的数据丢失,但是也带来了读写速度大大下降的副作用,也就是读出后,再次回写数据.

放一张对比图:

足以看出DRAM的鸡肋.

通常来说,DRAM里的栅极电容内的电荷只能维持2ms,就算不断电,2ms后存储的信息也会流失.

DRAM的刷新

为了解决DRAM栅极电容带来的缺点,我们给DRAM的电容进行刷新操作.
那么:
刷新周期: 2ms
每次刷新数量: 以行为单位,每次刷新一行存储单元

之所以以行为单位来刷新,是因为如果存储单元数量巨大,比如1M,那就有1M根选通线从译码器连出,工程量巨大,效率低下.
如果我们更改存储单元排列结构,将一维排布的存储单元转化为二位排列

使用两个译码器,一个行地址译码器,一个列地址译码器,这样可以优化存储单元排布。
一维的方案,译码器要处理 2^20 (20根地址线)的信息,转到二维后每个译码器只需要处理2^10的信息,即1k根选通线

一维转成二维后,对地址的解释也会有所变化,比如对于一维来说,0000 0000,表示的是第0个存储单元,而对于二维来说,前四个0表示行后四个表示列,只有被行,列地址同时选中的存储单元才能被读写.

刷新方法:这里且不讨论硬件实现,读出一行的信息后重新写入,占用1个读/写周期
在什么时刻刷新?
假设DRAM的内部结构排列成128 * 128,存取周期0.5us,那么2ms的极限时间就有2ms/0.5us = 4000个周期
也就是说2ms可以执行4000个周期的读写操作

思路一:每次读写完都刷新一行,也就是说,每次读写,用0.5us读写,0.5us刷新,系统的存取周期变1us--分散刷新


那么在2ms内我们可以刷新2ms / 1us = 2000次,而我们的DRAM阵列假设只有128行所以绰绰有余.
看起来很低效

思路二:2ms内集中安排时间全部刷新,那么系统的存取周期还是0.5us,2ms内划出一段时间来刷新,这个段时间无法访问存储器,这段时间被称为访存死区 -- 集中刷新


因为只有128行,所以我们只需要留下128个读写周期
看起来也好不到哪去

思路三:2ms内每行刷新一次即可,2ms内需要产生128次刷新请求,我们可以每隔2ms/128 = 15.6us一次,也就是说,每15.6us会有0.5us的死区 -- 异步刷新


而且我们可以选择在译码阶段我们进行刷新
看起来高效了很多

DRAM的地址线复用技术

前面提到,我们把一维的存储单元排布转换成了二维.

寻址时,如果我们同时把行地址列地址同时送到行地址译码器和列地址译码器,那么我们便需要相对应的地址线.比如0000 0000这一个地址会同时送到行地址译码器,列地址译码器,也就是说地址线要8根.

也就是说,我们的地址有多少位,我们就需要多少根地址线,这看起来理所当然,但我们有更好的方案,如果地址线一多,这种一次送到的方法就会显得鸡肋了.

我们可以先传行地址再传列地址或者反过来,这样如果是8位地址,我们每次只需要传4位,也就是地址线数量减半,芯片引脚地址线减半

SRAM的容量较小,所以送地址的时候都是同时送,也就是不分批.

ROM

Read Only Memory--只读存储器,断电后能持续保存数据

MROM(Mask Read-Only Memory) --掩模式只读存储器
厂家按照客户需求,在生产过程中直接写入信息,之后任何人不能重写,可靠性高,灵活性极低,生产周期长


PROM(Programmable Read-Only Memory) --可编程只读存储器
用户可用专门的PROM写入器来写入信息,一旦写入不可更改


EPROM(Erasable Programmable Read-Only Memory) --可擦可编程只读存储器
允许用户写入信息,并用特定方式擦除数据,可以多次重写


UVEROM(Ultraviolet rays Read-Only Memory) --可擦可编程只读存储器
允许用户写入信息,用紫外线照射一定时间来擦除数据.


EEPROM(Electrically Erase Programmable Read-Only Memory) --电可擦除可编程只读存储器
可用电擦除,擦除特定字


Flash Memory -- 闪速存储器
由EEPROM发展而来,可进行多次快速擦除重写,像是u盘,SD卡
由于闪存要先擦再写,所以闪存的读会比写快得多.


SSD(Solid State Drives) --固态硬盘
由存储单元(Flash)+控制单元构成,可进行多次快速擦写.

PC主板上的BIOS程序写在ROM里,逻辑上主存由RAM + ROM组成,而且常常统一编址,开机后先读取BISO程序.

优化RAM速度

双端口RAM

先看回存储周期:

DRAM的恢复时间比较长,有可能是存取的几倍.
虽然存取的时间很快,但是存取完要相当长的恢复时间,对于多核CPU来说对性能削弱是毁灭性的.

为了实现双端口RAM,我们要有两组完全独立的数据线,地址线,控制线.CPU,RAM里也要有更复杂的控制电路

CPU1,CPU2对双端口RAM访问的可能情况:
1.两个端口同时对不同的地址存取数据 √
2.两个端口同时对同一地址读出数据 √
3.两个端口同时对同一地址写入数据 ×
4.两个端口对同一地址单元,一个写入,一个读出 ×

解决方法: 使用"忙"信号,由判断逻辑决定暂时关闭一个端口 -- 即被延时访问

感觉很低效.

多体并行存储器

每一列可以看成一根内存

每个模块都有相同的容量和存取速度,各模块都有独立的读写控制电路,地址寄存器和数据寄存器,他们既能并行工作又能交叉工作

高位交叉编址


CPU是通过地址来向访存的,高位交叉编址是指,在地址最高位留出一定的位置来区分访问哪一根内存,
图中有四个内存,假如8位地址,那么可以留出最前面的两位来确定访问哪一个根内存.

假设每条内存有8个存储单元,那么有4 * 8 = 32个存储单元,也就是2 ^ 5个,也就是可以用5bit来作为地址,
那么对于第一根内存条有00 000,后面三个000表示体内地址

可以看出,地址编排方式是竖着编的

假如说,每个存储体存储周期为T,存取时间为r假设T = 4r,那么恢复时间为3r.
对于高位交叉编址,连续访问:00000,00001,00010,00011,00100

可以看到压力全给到了M0内存
可以算出连续存取n个字,耗时nT,这里是5T.

低位交叉编址

同高位交叉编址,不过是用最低位来表示第几根内存

可以看出,地址编排方式是横着编的

假如说,每个存储体存储周期为T,存取时间为r假设T = 4r,那么恢复时间为3r.
对于低位交叉编址,连续访问:00000,00001,00010,00011,00100

可以看到负载均匀,在存取00000后,直接去存取00001不用等待M0的恢复,只花了r时间,如此往下,
M3开始,M0刚好恢复,所以可以直接存取00100.有多线程的味道了.

图里划虚线的地方刚好是T,如果考虑上最后的恢复时间,那么总耗时是T + 4r
如果只考虑存取的总用时,那么只有5r

那么可以得出通用计算公式: 连续取n个存储字 耗时 = nr + 3r = T + (n -1)r (包含了最后恢复时间)
根据公式来看,当连续读写的字越多,时间越接近r

可以看出低位交叉编址比高位交叉编址高效的多的多.

为什么是"连续访问"
不碰到跳转语句的话,程序是一句一句顺序的执行的,亦或者像是数组这种结构数据紧密相连.

采用"流水线“(看看上面存储体时间图),看起来宏观上是并行的,微观上是串行的.宏观上,一个存储周期内,m体交叉存储器(m体指的是m个存储体,比如m条内存条),可以朝外提供的数据量为单个模块的m倍

存储周期为T,存取时间为r(或总线传输周期为r -- 指cpu通过数据总线传输数据至少要r的时间)
,为了使得流水线不间断,我们应该保证模块数m(内存条数) > T / r
原因如下:

如果给出一个地址x,我们可以通过体号(低位的那几个)来判断在那个存储体,也可以直接用x % m,m指的是存储体总量.

双通道内存使用的就是低位交叉的多体存储器

单体多字存储器

多体并行存储器的合并版.

整个存储器只有一套读写电路
本来每次只能读写一个字,但是到了这里的单体多字存储器一次能读4个字也就是一行.

显然,一次只能读4个字就显示出了他的鸡肋 -- 不能单独取其中的某个字,假如数据存在第一行后面两个字和第二行前面两个字,那么要读两次,而且会出来4个字的无效数据,而且指令和数据在主存内都是连续存放的.

但是吧,单体多字存储器确实提升了速度,每一个存取周期能读4个字
